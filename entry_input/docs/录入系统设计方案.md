# 个人信息录入系统设计方案

## 一、系统定位

### 目标

稳定、简单、用户友好的记录录入系统。

### 原则

- 原始数据保持简单干净
- 用户可直接查看和编辑
- 与分析系统完全解耦
- 支持细粒度版本控制
- 支持附件管理
- 支持安全备份

---

## 二、目录结构

```
raw/
└── 2026/
    ├── 20260301/
    │   ├── 20260301.md              # 当日索引
    │   ├── 20260301_122334.md       # 单条记录
    │   ├── 20260301_153012.md
    │   └── attachment/              # 当日附件目录
    │       ├── 20260301_122334_a1b2c3d4.jpg
    │       └── 20260301_153012_e5f6g7h8.pdf
    ├── 20260302/
    │   ├── 20260302.md
    │   ├── 20260302_091523.md
    │   └── attachment/
    └── 20260303/
        └── ...
```

### 规则

| 类型 | 格式 | 说明 |
|-----|------|------|
| 年份目录 | `YYYY` | 如 `2026` |
| 日期目录 | `YYYYMMDD` | 如 `20260301` |
| 当日索引 | `YYYYMMDD.md` | 如 `20260301.md` |
| 单条记录 | `YYYYMMDD_HHMMSS.md` | 如 `20260301_122334.md` |
| 附件目录 | `attachment/` | 位于日期目录下 |
| 附件文件 | `YYYYMMDD_HHMMSS_UUID.类型` | 如 `20260301_122334_a1b2c3d4.jpg` |

## 三、单条记录格式

### 文件命名

`YYYYMMDD_HHMMSS.md`

示例：`20260301_122334.md`

### 文件内容

纯文本，无 YAML front matter。

**普通记录示例：**

```
#todo #project-A this project is late for timeline.
```

**带附件的记录示例,用Obsidian内联的语法记录附件：**

```
#todo #project-A 拍了一张设计草图：

[[attachment/20260301_122334_a1b2c3d4.jpg]]
```

### 内容约定

| 字段 | 来源 | 解析方式 |
|-----|------|---------|
| created | 文件名 | `20260301_122334` → `2026-03-01 12:23:34` |
| tags | 正文 | 提取所有 `#xxx`，排除 `#project-xxx` |
| project | 正文 | 提取 `#project-xxx`，取 `xxx` 部分 |
| content | 正文 | 去掉所有 `#xxx` 后的纯文本 |

## 四、当日索引格式

### 文件命名

`YYYYMMDD.md`

### 文件内容

```markdown
# 2026-03-01 周六

---

![[20260301_122334]]

---

![[20260301_153012]]
```

### 维护规则

- 新增记录时，自动追加到当日索引
- 删除记录时，自动从当日索引移除

## 五、附件管理

### 目录位置

`YYYYMMDD/attachment/`

### 文件命名

`YYYYMMDD_HHMMSS_8位UUID.原文件类型`

示例：
- `20260301_122334_a1b2c3d4.jpg`
- `20260301_122334_e5f6g7h8.mp3`
- `20260301_122334_12345678.pdf`

### 内联语法

使用 Markdown 标准语法：

| 类型 | 语法 | 说明 |
|-----|------|------|
| 图片 | `[[path.jpg]]` | 可直接渲染 |
| 视频 | `[[path.mp4]]` | 部分编辑器支持 |
| 音频 | `[[path.mp3]]` | 链接形式 |
| 文档 | `[[path.pdf]]` | 链接形式 |

### 支持类型

图片、视频、音频、文档等。

## 六、Git 管理

### 初始化

```bash
cd raw/
git init
git add .
git commit -m "init: 初始化录入系统"
```

### 自动提交时机

- 新增记录后自动提交
- 修改记录后自动提交（包括更新附件名称、链接，删除链接等操作）
- 删除记录后自动提交
- 新增附件后自动提交

### 提交格式

```bash
git add .
git commit -m "add: 20260301_122334"
```

### 版本控制优势

| 功能 | 说明 |
|-----|------|
| 历史追溯 | 查看每条记录的创建和修改历史 |
| 误删恢复 | 通过 git 恢复已删除的记录 |
| 变更对比 | 对比记录内容的修改差异 |
| 细粒度追踪 | 每条记录独立追踪，互不影响 |

## 七、配置管理

### 配置文件

`config.yaml`

```yaml
raw_path: ./raw                     # raw 库路径（相对或绝对路径）
git_auto_commit: true               # 是否自动 git commit
git_auto_push: false                # 是否自动推送到远程仓库
git_remote_url: git@github.com:xxx/private-entries.git  # 私有远程仓库地址

backup:
  enabled: true                     # 是否启用加密备份
  tool: restic                      # 备份工具：restic 或 borg
  repo_path: ./backup-repo          # 备份仓库路径
  schedule: daily                   # 备份频率：daily 或 weekly
```

### 初始化流程

```
1. 检查 config.yaml 是否存在
   ├── 不存在 → 询问用户 raw 库位置
   │           ├── 用户输入路径（或使用默认 ./raw）
   │           ├── 创建 config.yaml 保存配置
   │           └── 初始化 raw 库（创建目录 + git init）
   └── 存在 → 读取配置，继续执行
```

### 交互示例

```bash
$ input_entry.py add "第一条记录"

欢迎使用录入系统！

请指定 raw 库存放位置（默认: ./raw）: 
> /Users/xxx/my-notes/raw

已创建配置文件: config.yaml
已初始化 raw 库: /Users/xxx/my-notes/raw
已完成 git 初始化

✓ 已创建记录: 20260301_122334.md
```

### 路径更新

```bash
# 命令方式更新
input_entry.py config --raw-path /new/path/to/raw

# 启动时自动检测
$ input_entry.py add "新记录"

警告: raw 库路径不存在
当前配置: /Users/xxx/my-notes/raw

请输入新的 raw 库路径:
> /Users/xxx/new-location/raw

配置已更新，继续执行...
```

## 八、备份机制

### 备份策略

| 备份类型 | 工具 | 频率 | 存储位置 | 说明 |
|---------|------|------|---------|------|
| 版本控制 | Git 本地 | 每次变更 | 本地 | 误删恢复、变更对比 |
| 远程同步 | Git 远程 | 每次推送 | 私有仓库 | 异地备份、多设备同步 |
| 加密快照 | Restic/Borg | 每日/每周 | 本地/云盘/外置硬盘 | 增量备份、加密、去重 |

### Git 远程仓库

**注意事项：**
- 不要推送到公开仓库（含个人隐私数据）
- 使用私有仓库（GitHub Private / GitLab / Gitea 等）

```bash
# 配置远程仓库
cd raw/
git remote add origin git@github.com:xxx/private-entries.git
git push -u origin main
```

### 加密增量备份工具

#### Restic（推荐）

**特点：**
- 跨平台（Mac/Linux/Windows）
- 增量备份 + 加密 + 去重
- 单二进制文件，无依赖
- 支持多种存储后端

**安装：**
```bash
# Mac
brew install restic

# Linux
apt install restic

# Windows
scoop install restic
```

**使用：**
```bash
# 初始化加密仓库
restic init --repo /path/to/backup-repo

# 备份（增量）
restic backup --repo /path/to/backup-repo /path/to/raw

# 查看快照
restic snapshots --repo /path/to/backup-repo

# 恢复
restic restore --repo /path/to/backup-repo latest --target /path/to/restore

# 清理旧备份（保留最近7天）
restic forget --repo /path/to/backup-repo --keep-daily 7 --prune
```


### 定时自动备份

```bash
# cron 每日备份（Restic）
0 2 * * * RESTIC_PASSWORD="xxx" restic backup --repo /backup-repo /path/to/raw && restic forget --repo /backup-repo --keep-daily 7 --prune >> /var/log/backup.log 2>&1
```

## 十二、Agent 安全约束设计

### 设计背景

本系统设计为 AI Agent 助手的个人信息管理工具，考虑到 Agent 的自动化特性，必须在系统层面设计安全约束，保护用户数据安全。

### 约束一：禁止删除

**设计决策：Agent 无删除权限**

| 层面 | 实现方式 |
|-----|---------|
| 文档约束 | SKILL.md 和 AGENTS.md 明确禁止 Agent 调用 delete 命令 |
| 命令保留 | delete 命令保留，仅供用户直接操作使用 |
| 响应策略 | Agent 遇到删除请求时，回复"没有删除权限，请手动删除" |

**理由：**
- 用户数据不可逆，Agent 误删无法恢复
- Git 虽然可以恢复，但增加用户负担
- 用户手动删除更安全，有确认机会

### 约束二：禁止篡改

**设计决策：Agent 必须原样保存用户输入**

| 层面 | 实现方式 |
|-----|---------|
| 文档约束 | SKILL.md 和 AGENTS.md 明确禁止 Agent 添加或修改内容 |
| 标签处理 | Agent 不自动添加标签，仅用户自己在输入中指定 |
| 内容处理 | add 命令的内容参数必须一字不差地保存用户原始输入 |

**理由：**
- 保持用户原始记录的完整性
- Agent 可能误解用户意图，自动添加内容会造成混乱
- 用户自己编辑更可靠，知道自己写了什么

### 约束三：用户可编辑

**设计决策：原始数据保持简单干净**

| 层面 | 实现方式 |
|-----|---------|
| 格式选择 | Markdown 纯文本，无 YAML front matter |
| 文件位置 | 用户可直接访问 raw/ 目录 |
| 编辑方式 | 用户可用任意文本编辑器直接修改文件 |
| 版本控制 | Git 追踪所有变更，支持历史回溯 |

**理由：**
- 用户随时可以绕过 Agent 直接操作
- 纯文本格式易于查看和编辑
- Git 提供安全网，误操作可恢复

## 十三、设计要点总结

| 要点 | 决策 |
|-----|------|
| 存储格式 | Markdown，纯文本 |
| 时间记录 | 文件名包含时间戳，内容无 YAML |
| 标签方式 | 正文用 `#xxx` |
| 项目归属 | `#project-xxx` 格式 |
| 当日索引 | Obsidian `![[ ]]` 嵌入语法 |
| 目录层级 | 2 层：年/日期 |
| 附件目录 | `YYYYMMDD/attachment/` |
| 附件命名 | `YYYYMMDD_HHMMSS_8位UUID.类型` |
| 附件语法 | Markdown 标准 `![](path)` |
| Git 管理 | 每条记录/附件变更后自动提交 |
| Git 远程 | 推送私有仓库，不推送公开仓库 |
| 配置文件 | `config.yaml`，保存 raw 库路径 |
| 初始化 | 首次启动询问用户，创建配置 + 初始化 raw 库 |
| 路径更新 | 命令更新或启动时自动检测 |
| 本地备份 | Git 本地版本控制 |
| 远程备份 | Git 推送私有远程仓库 |
| 加密备份 | Restic（增量 + 加密 + 去重） |
| 队列系统 | persist-queue，CRUD 时自动入队 | **第一版不引入，待录入系统稳定后再加入**
| ID 方案 | 无额外 ID，时间戳即可 |

## 十四、示例操作流程

### 新增记录（无附件）

```bash
$ input_entry.py add "#todo #project-A this project is late for timeline."
```

**系统执行：**
1. 获取当前时间：`2026-03-01 12:23:34`
2. 创建目录：`raw/2026/20260301/`
3. 创建文件：`raw/2026/20260301/20260301_122334.md`
4. 写入内容：`#todo #project-A this project is late for timeline.`
5. 更新索引：`raw/2026/20260301/20260301.md`（追加嵌入引用）
6. 入队：`queue.put({file: "...", action: "upsert"})`
7. Git 提交：`git commit -m "add: 20260301_122334"`

### 新增记录（带附件）

```bash
$ input_entry.py add "#todo 拍了一张设计草图" --attach ~/Downloads/photo.jpg
```

**系统执行：**
1. 获取当前时间：`2026-03-01 12:23:34`
2. 生成附件名：`20260301_122334_a1b2c3d4.jpg`
3. 复制附件到：`raw/2026/20260301/attachment/`
4. 创建记录文件，追加附件引用：`![](attachment/20260301_122334_a1b2c3d4.jpg)`
5. 更新索引 + 入队 + Git 提交
