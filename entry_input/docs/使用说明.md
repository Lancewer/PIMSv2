# Entry Input Skill 使用说明

## 快速开始

### 1. 初始化系统

首次使用时，需要初始化系统：

```bash
python ~/devspace/pims_v2/skills/entry_input/index.py init
```

系统会：
- 创建数据目录 `raw/`
- 初始化 Git 仓库
- 生成配置文件 `config.yaml`

### 2. 添加记录

```bash
# 基础添加
python ~/devspace/pims_v2/skills/entry_input/index.py add "这是一条记录"

# 带标签（用户自己指定）
python ~/devspace/pims_v2/skills/entry_input/index.py add "#todo 完成报告"

# 带附件
python ~/devspace/pims_v2/skills/entry_input/index.py add "拍了一张照片" --attach ~/Downloads/photo.jpg
```

### 3. 查看记录

```bash
# 查看今日记录
python ~/devspace/pims_v2/skills/entry_input/index.py list today

# 查看指定日期
python ~/devspace/pims_v2/skills/entry_input/index.py list 2026-04-14

# 查看所有记录
python ~/devspace/pims_v2/skills/entry_input/index.py list

# 查看单条记录详情
python ~/devspace/pims_v2/skills/entry_input/index.py show 20260414_120000
```

## 命令详解

### init - 初始化

```bash
python index.py init
```

首次运行时自动创建：
- `raw/` 数据目录（按年份和日期分层）
- Git 仓库（自动追踪所有变更）
- `config.yaml` 配置文件

### add - 添加记录

```bash
# 基础用法
python index.py add "记录内容"

# 带附件
python index.py add "记录内容" --attach /path/to/file
```

**附件处理流程：**
1. 自动复制文件到当日 `attachment/` 目录
2. 自动重命名为 `YYYYMMDD_HHMMSS_UUID.类型`
3. 自动在记录中添加引用 `[[attachment/xxx]]`

**记录内容约定：**
- 纯文本，无 YAML
- 使用 `#标签` 标记分类（如 `#todo`、`#memo`）
- 使用 `#project/xxx` 标记项目归属
- 内容会被原样保存，不添加任何额外内容

### list - 查看列表

```bash
# 今日
python index.py list today

# 指定日期
python index.py list 2026-04-14

# 所有记录
python index.py list
```

输出格式：
```
共 3 条记录:
  20260414_090000: 完成报告...
  20260414_100000: #todo 修改文档...
  20260414_110000: 拍了一张照片...
```

### show - 查看详情

```bash
python index.py show 20260414_090000
```

输出格式：
```
--- 20260414_090000 ---
完成报告

[[attachment/20260414_090000_abc12345.pdf]]
```

### edit - 编辑记录

```bash
# 添加附件到已有记录
python index.py edit 20260414_090000 --attach /path/to/new_file.pdf
```

> ⚠️ **注意**：Agent 禁止修改记录内容，仅用户可手动编辑文件。

### delete - 删除记录

```bash
python index.py delete 20260414_090000
```

> 🚫 **Agent 禁止执行删除操作**。此命令仅供用户直接使用。

### config - 配置管理

```bash
# 查看当前配置
python index.py config

# 更新数据路径
python index.py config --raw-path /new/path/to/raw

# 设置 Git 远程仓库
python index.py config --git-remote git@github.com:xxx/private.git
```

### git-push - 推送远程

```bash
python index.py git-push
```

将数据推送到配置的 Git 远程仓库。

### backup - 加密备份

```bash
# 执行备份
python index.py backup

# 查看备份快照列表
python index.py backup --list

# 恢复备份
python index.py backup --restore latest --target /path/to/restore
```

使用 Restic 进行加密增量备份。

## 数据结构

### 目录结构

```
raw/
└── 2026/                          # 年份目录
    └── 20260414/                  # 日期目录
        ├── 20260414.md            # 当日索引
        ├── 20260414_090000.md     # 单条记录
        ├── 20260414_100000.md
        └── attachment/            # 附件目录
            ├── 20260414_090000_abc12345.pdf
            └── 20260414_100000_def67890.jpg
```

### 文件命名

| 类型 | 格式 | 示例 |
|-----|------|------|
| 记录文件 | `YYYYMMDD_HHMMSS.md` | `20260414_090000.md` |
| 附件文件 | `YYYYMMDD_HHMMSS_UUID.类型` | `20260414_090000_abc12345.pdf` |
| 当日索引 | `YYYYMMDD.md` | `20260414.md` |

### 记录格式

**普通记录：**
```
#todo #project/xxx 完成设计文档
```

**带附件的记录：**
```
#todo 拍了一张设计草图：

[[attachment/20260414_090000_abc12345.jpg]]
```

### 当日索引格式

```markdown
# 2026-04-14 周一

---

![[20260414_090000]]

---

![[20260414_100000]]
```

使用 Obsidian 嵌入语法，可在 Obsidian 中直接预览。

## 标签约定

| 标签 | 含义 | 示例 |
|-----|------|------|
| `#todo` | 待办事项 | `#todo 完成报告` |
| `#memo` | 备忘 | `#memo 记得买牛奶` |
| `#idea` | 想法 | `#idea 新项目构思` |
| `#note` | 笔记 | `#note 学习笔记` |
| `#project/xxx` | 项目归属 | `#project/pims 完成功能` |
| `#xxx` | 自定义标签 | `#work 会议记录` |

> ⚠️ 标签由用户自己在输入中指定，系统不会自动添加。

## 配置说明

配置文件路径：`~/devspace/pims_v2/skills/entry_input/config.yaml`

```yaml
raw_path: ./raw                     # 数据存储路径
                                    # 建议使用绝对路径，如：
                                    # /Users/xxx/Documents/pims/raw

git_auto_commit: true               # 每次变更自动提交
git_auto_push: false                # 是否自动推送（建议手动）
git_remote_url: ''                  # 远程仓库地址
                                    # 示例：git@github.com:xxx/private-pims.git

backup:
  enabled: true                     # 启用加密备份
  tool: restic                      # 备份工具（restic 或 borg）
  repo_path: ./backup-repo          # 备份仓库路径
  schedule: daily                   # 备份频率
```

## 安全提示

### Git 版本控制

- 所有变更自动提交到本地 Git
- 可通过 `git log` 查看历史
- 可通过 `git checkout` 恢复误删文件

### 远程备份

- 推送前确保远程仓库为**私有仓库**
- 不要推送到公开仓库（数据含个人信息）

### 加密备份

- 使用 Restic 进行加密增量备份
- 备份密码请妥善保管，丢失无法恢复

## Agent 安全约束

本系统设计为 AI Agent 的个人信息管理工具，内置以下安全约束：

### 🚫 禁止删除

- Agent 无删除权限
- delete 命令仅供用户直接操作
- 用户需要手动删除数据文件

### 🚫 禁止篡改

- Agent 必须原样保存用户输入
- 不自动添加标签
- 不修改用户内容

## 常见问题

### Q: 如何修改已创建的记录？

**A:** 直接编辑对应的 `.md` 文件。文件路径：`raw/YYYY/YYYYMMDD/YYYYMMDD_HHMMSS.md`

### Q: 如何删除记录？

**A:** 直接删除对应的 `.md` 文件和附件。系统会自动更新当日索引。

### Q: 如何迁移数据到其他设备？

**A:** 
1. 配置 Git 远程仓库
2. 执行 `python index.py git-push`
3. 在新设备上 `git clone` 仓库

### Q: 如何恢复误删的记录？

**A:** 
1. 进入 raw 目录
2. 执行 `git log` 查看历史
3. 执行 `git checkout <commit> -- <file>` 恢复

### Q: 附件文件太大怎么办？

**A:** 系统不限制附件大小。但建议：
- 大文件使用外部存储（如云盘）
- 记录中添加链接而非附件
